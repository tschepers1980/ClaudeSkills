# Category 5: Mission Critical & Customer Data Vendors

## Critical Requirements (All Mandatory)

### Certifications & Compliance
- **ISO 27001** AND **SOC2 Type II** required
- **NEN 7510** for healthcare-related services
- **GDPR/AVG full compliance with documentation**
- **DPIA (Data Protection Impact Assessment)** required
- **Annual third-party security audits**

### Advanced Security Controls

#### Security Frameworks
- Implementation of NIST/CIS/Microsoft Secure Score
- Information Security Management System (ISMS)
- Security Operations Center (SOC) 24/7
- SIEM implementation with active monitoring

#### Network Security
- Network segmentation mandatory
- Zero-trust architecture preferred
- Managed EDR/XDR deployment
- Regular vulnerability scanning (monthly)

#### Testing & Validation
- Penetration testing (annual minimum)
- Code security reviews (SAST/DAST)
- Red team exercises
- Disaster recovery testing (quarterly)

### Detailed Assessment Areas

#### 1. Business Continuity
- Business Continuity Plan (BCP) required
- Recovery Time Objective (RTO) < 4 hours
- Recovery Point Objective (RPO) < 1 hour
- Annual BCP testing with documentation
- Geographically distributed backups

#### 2. Advanced Access Control
- Privileged Access Management (PAM)
- Just-in-time access provisioning
- Separation of duties enforced
- Comprehensive audit logging
- User behavior analytics

#### 3. Data Governance
- Data classification scheme
- Data lifecycle management
- Encryption key rotation
- Data residency guarantees
- Exit strategy documented

#### 4. Incident Management
- Incident response team 24/7
- Notification within 4 hours
- Forensic capabilities
- Breach insurance coverage
- Regular tabletop exercises

### Required Documentation
- All Category 4 documents PLUS:
- DPIA assessment results
- BCP documentation
- Network architecture diagrams
- Security roadmap
- Compliance attestations
- Financial stability reports
- Insurance policies
- Exit strategy plan

### Monitoring Requirements
- Include in quarterly IT reports
- Monthly security metrics review
- Real-time monitoring dashboards
- Automated compliance checking
- Regular executive briefings

### Specific Questions

1. **Organizational Security**
   - CISO reporting structure?
   - Security team size and expertise?
   - Security budget as % of revenue?
   - Board-level security oversight?

2. **Technical Architecture**
   - Multi-tenancy isolation methods?
   - Data encryption in memory?
   - API security controls?
   - Container/cloud security measures?

3. **Compliance & Legal**
   - Litigation history (data/security)?
   - Regulatory compliance status?
   - Cyber insurance coverage amount?
   - Indemnification clauses?

4. **Supply Chain**
   - Fourth-party risk management?
   - Critical supplier assessment?
   - Software bill of materials (SBOM)?
   - Open source vulnerability management?

### DPIA Triggers
Conduct DPIA when:
- Processing special category data
- Large-scale systematic monitoring
- Automated decision-making with legal effects
- Processing children's data
- Cross-border data transfers
- New technologies with privacy impact

### Exit Strategy Requirements
- Data export in standard formats
- Transition assistance period
- Data deletion certification
- Knowledge transfer provisions
- Escrow arrangements for critical code

### Escalation Criteria
Immediate board notification for:
- Security certification lapses
- Data breach incidents
- Compliance violations
- Financial instability
- Service degradation patterns
- Unmitigated critical risks
