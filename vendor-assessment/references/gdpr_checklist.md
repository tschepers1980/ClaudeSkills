# GDPR Compliance Checklist

## Legal Basis & Documentation

### Data Processing Agreement (DPA)
- [ ] DPA available and signed
- [ ] Includes all required GDPR articles
- [ ] Sub-processors listed and approved
- [ ] Liability and indemnification clauses present
- [ ] Breach notification procedures defined (72 hours)

### Legal Basis for Processing
- [ ] Legitimate interest assessment completed
- [ ] Consent mechanisms in place (if applicable)
- [ ] Contract necessity documented
- [ ] Legal obligation identified (if applicable)

## Data Subject Rights

### Rights Implementation
- [ ] Right to access (data export functionality)
- [ ] Right to rectification (data correction)
- [ ] Right to erasure (data deletion)
- [ ] Right to portability (standard formats)
- [ ] Right to restrict processing
- [ ] Right to object

- [ ] Response timeline â‰¤ 30 days
- [ ] Identity verification process
- [ ] Automated request handling available
- [ ] Manual override capabilities

## Privacy by Design

### Data Minimization
- [ ] Only necessary data collected
- [ ] Retention periods defined
- [ ] Automatic deletion implemented
- [ ] Purpose limitation enforced

### Security Measures
- [ ] Encryption at rest (AES-256 minimum)
- [ ] Encryption in transit (TLS 1.2+)
- [ ] Access controls (RBAC)
- [ ] Audit logging
- [ ] Regular security testing

## International Transfers

### Transfer Mechanisms
- [ ] Data stays within EEA
- [ ] OR Standard Contractual Clauses (SCCs) in place
- [ ] OR Adequacy decision applicable
- [ ] OR Binding Corporate Rules (BCRs)
- [ ] OR Specific derogations apply

### Additional Safeguards
- [ ] Transfer Impact Assessment (TIA) completed
- [ ] Supplementary measures implemented
- [ ] Local law assessment performed
- [ ] Government access risks evaluated

## Accountability

### Documentation
- [ ] Privacy policy publicly available
- [ ] Internal processing records maintained
- [ ] Data flow mapping completed
- [ ] Risk assessments documented
- [ ] Training records kept

### Governance
- [ ] DPO appointed (if required)
- [ ] Privacy team identified
- [ ] Regular compliance reviews
- [ ] Management oversight established

## Breach Management

### Incident Response
- [ ] Breach detection capabilities
- [ ] Classification procedures (high risk?)
- [ ] 72-hour authority notification
- [ ] Data subject notification (if high risk)
- [ ] Breach register maintained

### Mitigation Measures
- [ ] Encryption reduces notification requirement
- [ ] Pseudonymization implemented
- [ ] Backup and recovery procedures
- [ ] Business continuity planning

## Special Categories

### Sensitive Data (if applicable)
- [ ] Health data protections
- [ ] Biometric data safeguards
- [ ] Children's data considerations
- [ ] Criminal conviction data controls

### High-Risk Processing
- [ ] DPIA conducted
- [ ] Consultation with DPA (if required)
- [ ] Risk mitigation measures implemented
- [ ] Regular DPIA reviews scheduled

## Vendor-Specific Requirements

### Category 3 Vendors
- Basic GDPR awareness confirmed
- Data location identified

### Category 4 Vendors
- Full GDPR compliance verified
- DPA mandatory
- Sub-processor approval required

### Category 5 Vendors
- Comprehensive GDPR audit
- DPIA mandatory
- Regular compliance monitoring
- Exit strategy includes data deletion
